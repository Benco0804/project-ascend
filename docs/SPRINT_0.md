# Project Ascend – Sprint 0

## Goal

Define the core product foundation before writing code.

Sprint 0 should answer:

- Who is the first user?
- What problem are we solving?
- What is the core loop?
- What behavior are we trying to create?
- What goes into MVP?
- What stays out?
- How do we measure success?

---

## Sprint 0 Outputs

1. Vision
2. First target user
3. Core problem
4. Core loop
5. MVP scope
6. Success metrics
7. Feature kill list
8. Technical direction

---

## Working Rule

No code before the core loop is clear.

A feature only enters MVP if it improves:

- retention
- habit formation
- daily engagement
- user progress
